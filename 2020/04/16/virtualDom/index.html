什么是虚拟DOM

Virtual Dom本质上就是js 对象，这个对象是对real Dom的描述。

优缺点：

  • 局部更新，批处理可以保证比较高效的渲染和比较少的性能损耗

  • 跨平台，由于是js对象，既可以用于浏览器，也可以用于SSR

  • 首次渲染大量DOM时，由于多了一层虚拟DOM计算，会比真实DOM渲染慢

综上，对于dom操作比较少，仅仅是展示类的页面使用真实DOM更高效，如果有大量的DOM操作使用虚拟DOM更能提高性能。

  1. Virtual Dom 算法

Virtual Dom的优点之一是局部更新，就是因为diff算法。两颗树的完全的 diff 算法时间复杂度为 O(n^3) 。但是在前端当中，你很少会跨越层级的移动DOM元素。所以 Virtual DOM diff算法的核心是对同一个层级的元素进行对比算法复杂度是O(n)。

  • 深度优先遍历，记录差异
    在对比的过程中，会对两颗树进行深度优先的遍历，这样每个节点都有一个唯一的标识，在深度遍历的过程中，每遍历到一个节点就对比该节点是否有差异，如果有差异就记录到一个对象中。
    而这个差异，有哪些呢？
  1. 节点替换
  2. 节点移动，移除，新增子节点
  3. 修改节点属性
  4. 文本节点内容变化

对于节点替换，我们只要对比tagName,不同就进行替换
对于节点移动，为了节点复用，我们希望每个节点都有一个独一无二的key，这样我们只要对key进行对比
至于如何移动，这个问题抽象出来其实是字符串的最小编辑距离问题，最长见的解题算法是Levenshtein Distance算法， 通过动态规划求解。

  • 渲染器patch
    渲染器就是将虚拟DOM映射到真实的DOM
